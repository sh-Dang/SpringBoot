[Eureka 사용 이유] - 규모가 클 때 사용

1. 서비스 주소 자동 관리
    MSA를 이루고있는 서비스마다 각각 IP가 다를 수 있고, 특정 서비스의 경우엔 인스턴스를 여러개 둘 수도 있다.
    스케일 아웃(수평확장) -> 인스턴스를 여러개 두는 것
    이때 Eureka는 서비스들의 주소(IP/Port)를 중앙에서 관리함
    따라서 GateWay나 다른 서비스가 특정 서비스를 호출하고자 할때, 변결된 ip/port를 일일이 적는 하드코딩없이
    서비스 이름만으로도 호출이 가능
    ex) http://localhost:7773/members
        lb://member-service/members

2. 로드 밸런싱 지원
    같은 서비스가 여러개의 Instance를 생성할 수 있는데, Eureka는 이 중 하나를 골라줌
    ex) MemberService가 두개 떠있을 경우 자동으로 로드밸런싱을 수행하여 적절한 인스턴스를 선택해 줌

3. 장애 허용
    Eureka는 HeartBeat(헬스체크)를 통해 서비스가 살아있는지 여부를 확인할 수 있는 능력이 있음
    죽은 서비스가 있다면 등록정보(Registry)에서 제거하여, 다른 서비스가 호출하지 않도록 해줌

4. 실행 순서
